# Enjeux du Cloud : 
## 1. Sécurité, Cybersécurité et Conformité
- Le passage au cloud élargit considérablement la surface d'attaque des systèmes d'information.
- Mauvaises configurations et facteurs humains : En 2025/2026, l'immense majorité des failles (accès non autorisés) proviennent d'erreurs humaines, comme des configurations de droits (IAM - Identity and Access Management) trop permissives ou des identifiants faibles. L'approche Zero Trust et le principe du moindre privilège deviennent incontournables.
- Complexité du multicloud : Gérer la sécurité de manière cohérente à travers plusieurs fournisseurs (AWS, Azure, etc.) complexifie la centralisation des logs (SIEM) et le monitoring. Les environnements de test et de développement sont souvent les maillons faibles ciblés par les attaquants.
- Gouvernance des données : Face aux régulations européennes (RGPD, IA Act) et aux risques d'espionnage industriel, savoir précisément où sont stockées les données, qui y accède et comment elles transitent est devenu un défi critique pour les DSI.

## 2. Le défi financier (FinOps)
- Si le modèle économique du paiement à l'usage (OPEX) est séduisant, il représente un piège pour les entreprises non préparées.
- Le syndrome du "Open Bar" : Sans pilotage précis, les développeurs peuvent facilement "spinner" (démarrer) des machines surdimensionnées ou oublier d'éteindre des environnements de test le week-end, faisant exploser la facture.
- L'imprévisibilité des coûts : Les modèles de tarification sont complexes (frais de transfert sortant de données / egress fees, coûts des requêtes API). Les entreprises doivent impérativement former des équipes FinOps pour auditer, taguer et rationaliser les ressources consommées.

## 3. La dépendance technologique (Vendor Lock-in)
- Le coût de la réversibilité : S'il est facile d'entrer dans le cloud, en sortir est plus difficile. Les fournisseurs cloud lient leurs clients en proposant des services propriétaires natifs très performants (bases de données managées, outils d'IA natifs). Une fois le code fortement couplé à ces services, changer de fournisseur demande une réécriture coûteuse.
- Solutions envisagées : Les architectes IT doivent jongler entre des approches "Cloud Agnostic" (en utilisant massivement des conteneurs comme Docker et Kubernetes pour rester portables) et l'utilisation des services managés qui accélèrent le développement.

## 4. Impact environnemental et Green IT
- Une consommation énergétique massive : En 2025/2026, l'essor de l'Intelligence Artificielle générative a fait exploser les besoins en puissance de calcul (GPU) et en refroidissement des datacenters, pesant lourdement sur les réseaux électriques.
- Objectifs RSE (Responsabilité Sociétale des Entreprises) : 
 Les entreprises sont désormais sous pression pour mesurer et réduire leur empreinte carbone numérique. Le défi pour les équipes IT est d'adopter des pratiques de Green Software Engineering (optimisation du code, extinction des ressources inutilisées, choix de datacenters alimentés en énergie renouvelable).

## 5. Pénurie de compétences et transformation de l'IT
- Évolution des métiers : L'administrateur système classique (qui gère des serveurs physiques) doit évoluer vers des métiers d'ingénierie Cloud, DevOps, FinOps et DevSecOps.
- Le défi de la formation : L'infrastructure s'écrit désormais sous forme de code (Infrastructure as Code - Terraform, Ansible). Le facteur humain est crucial : la difficulté pour les entreprises est de recruter et de former leurs équipes pour maîtriser la complexité croissante de ces nouvelles architectures

# Les modèles du Cloud :

## Le concept de XaaS (Anything as a Service)
- L'acronyme XaaS, qui signifie "Tout en tant que service", désigne le regroupement de l'ensemble des services informatiques, produits et technologies délivrés à la demande via Internet.
- Ce modèle modifie profondément le rôle de la DSI : on passe d'une logique de constructeur/mainteneur de serveurs à une logique de consommateur/intégrateur de services managés.

## 2. IaaS (Infrastructure as a Service)
- Le concept : Location de ressources d'infrastructure brutes (serveurs virtuels, stockage, réseau).
- Cas d'usage :
* Hébergement de machines virtuelles (VM).
* Mise en place d'un plan de reprise d'activité (DRaaS - Disaster Recovery as a Service).
* Lift and Shift : Migration d'une infrastructure "On-Premise" (sur site) vers le cloud sans modifier l'architecture de l'application.

- Avantages :
* Contrôle total sur les systèmes d'exploitation (OS).
* Flexibilité pour installer des logiciels spécifiques non supportés dans les modèles supérieurs.

- Défis IT :
* L'équipe IT reste entièrement responsable des mises à jour de l'OS, des correctifs de sécurité et de la gestion des accès (Identity and Access Management).

## 3. PaaS (Platform as a Service)
- Le concept : Fourniture d'un environnement de développement et de déploiement managé. Le fournisseur gère l'infrastructure sous-jacente (matériel, réseau, OS).
- Cas d'usage :
* Développement d'applications (serveurs web, bases de données managées comme Azure SQL ou Amazon RDS).
* Conteneurisation (Kubernetes as a Service).
- Avantages :
* Accélération du Time-to-Market : Les développeurs se concentrent uniquement sur le code et les données.
* Maintenance simplifiée, les mises à jour des serveurs et des bases de données sont gérées par le fournisseur.
- Défis IT :
* C'est à ce niveau que le risque de Vendor Lock-in (enfermement propriétaire) est le plus fort. Si le code est développé spécifiquement pour une base de données propriétaire (ex: AWS DynamoDB), la migration vers un autre cloud sera complexe.

## 4. SaaS (Software as a Service)
- Le concept : Application logicielle complète, prête à l'emploi et gérée entièrement par l'éditeur.
- Cas d'usage :
* Messagerie professionnelle (Microsoft 365, Google Workspace).
* CRM (Salesforce).
* Outils de visioconférence et de collaboration.
- Avantages :
* Déploiement immédiat, accessible depuis n'importe quel navigateur (idéal pour le télétravail).
* Aucune gestion d'infrastructure pour l'entreprise ; les correctifs et nouvelles fonctionnalités sont appliqués de manière transparente par l'éditeur.
- Défis IT :
* Personnalisation très limitée (on s'adapte à l'outil, l'outil ne s'adapte pas à l'entreprise).
* Risque du Shadow IT : N'importe quel service métier peut souscrire à un SaaS avec une carte bancaire, contournant la politique de sécurité et de gouvernance des données de la DSI.

## 5. Les services spécialisés (DaaS, MaaS, etc.)
- L'évolution du cloud a créé de multiples sous-catégories utiles à l'IT :

- DaaS (Desktop as a Service) : Bureaux virtuels hébergés (ex: Azure Virtual Desktop). Idéal pour sécuriser les postes de travail, car aucune donnée n'est stockée localement sur la machine de l'utilisateur.
- MaaS (Monitoring as a Service) : Externalisation de la surveillance de l'infrastructure pour centraliser les alertes et les logs (Dynatrace)
